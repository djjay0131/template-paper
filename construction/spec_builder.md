# Spec Builder — Design Document Template

Use this template when creating new design documents in `construction/design/`.

---

## Template

```markdown
# Design: [Title]

## Objective
_What is this design for? What problem does it solve?_

## Target Venue Alignment
- **Venue**: _Target conference/journal/funding agency_
- **Topic Area**: _Which topic area does this address?_
- **Relevance**: _Why is this relevant to the venue?_

## Background
_Context, prior work, and motivation. What literature supports this?_

## Proposed Approach
_Detailed description of the approach or section content._

## Key Contributions
_What is novel? What advances the state of the art?_

## Feasibility
- **Timeline**: _Can this be completed within the project timeline?_
- **Resources**: _What is needed (compute, data, personnel)?_
- **Risks**: _What could go wrong? Mitigations?_

## References
_Key papers and resources._

## Acceptance Criteria
- [ ] _Criterion 1_
- [ ] _Criterion 2_

## Status
- **Created**: YYYY-MM-DD
- **Last Updated**: YYYY-MM-DD
- **State**: Draft | Review | Approved | Superseded
```

---

## Usage

1. Copy template to `construction/design/<name>.md`
2. Fill in all sections
3. Review against submission requirements
4. Get approval before drafting content
